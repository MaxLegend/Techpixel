# Документация проекта Techpixel

**Версия:** 0.0.1 (Alpha)

## Обзор

Techpixel — это воксельный 3D-движок для игр, вдохновлённый архитектурой Minecraft. Проект возник из идеи использования больших языковых моделей (LLM) для создания полноценного игрового движка, что делает сложную разработку доступной для тех, кто не обладает глубокими знаниями программирования.

### Принцип «No-Code»

Разработка движка велась в течение 4 месяцев при строгом условии: **полный запрет на ручное написание кода.** Основной целью было использование LLM (Claude, GLM-5/5.1) для всех задач программирования при использовании языка Rust в качестве основы. Rust был выбран благодаря качественной документации и обширной экосистеме библиотек, что критически важно для создания точных и контекстно-зависимых промптов для ИИ.

## Технические характеристики и возможности

### 0. Конвейер рендеринга (Pipeline)

Движок использует строго определённую последовательность этапов рендеринга:
`Небо (Sky)` → `Геометрия мира` → `Геометрия сущностей` → `Прозрачные блоки` → `Контур (Outline)` → `Интерфейс (UI)`.

**Архитектура состояний:**
`push()` → `load()` → `start()` → `[update() / render() / build_ui()]` → `unload()`

### 1. Глобальное освещение и источники света

* **Системы:** Трассировка воксельных конусов (VCT) для глобального освещения.
* **Тени:** Воксельный рейкастинг по алгоритму DDA для построения карт теней.
* **Дополнительные возможности:**
  * Поддержка динамического освещения (например, источники света, прикреплённые к сущностям).
  * Этап рендеринга прозрачности с частичным затенением и цветными тенями (например, свет, проходящий через красное стекло).
  * Объёмные лучи (volumetric rays) и световые ореолы вокруг источников света.

### 2. Конфигурация и генерация мира

* **На основе JSON:** Гибкая система конфигурации позволяет изменять параметры движка в реальном времени, включая размеры чанков.
* **Бесконечная вертикальность:** Поддержка бесконечной копки и строительства. Движок использует многослойную систему генерации мира.
* **Конвейер биомов:**
  * *Географический слой:* Континенты, острова, горы.
  * *Климатический слой:* Климатические зоны по оси Z, уровень влажности.
  * *Геологический слой:* Магматические, метаморфические и осадочные регионы.
  * *Стратиграфический слой:* Преднастроенные слои глубины (трава, земля, глина, камень, бедрок).

* **Встроенный визуализатор:** Внутриигровой графический интерфейс позволяет настраивать параметры генерации мира в реальном времени, с функцией «Сохранить в JSON» для фиксации настроек.

### 3. Управление блоками

* **Редактор блоков:** Выделенный внутриигровой интерфейс для создания и модификации блоков.
* **PBR и текстурирование:** Поддержка PBR-материалов, свечения (emission) и продвинутого смешивания текстур (texture splatting) — до 5 слоёв семплирования, что позволяет накладывать текстуры друг на друга (например, траву поверх земли).

### 4. Инфраструктура движка и оптимизация

* **Оптимизация:**
  * Отсечение по пирамиде видимости (frustum culling) на уровне чанков и геометрии.
  * Многопоточная загрузка и выгрузка чанков.
  * Приоритетный рендеринг по направлению.
  * 3-уровневая система детализации (LOD).
* **Отладка:** Комплексный мониторинг во время выполнения через `F3` (стандартная отладка) и `F4` (профайлер).
* **Поддержка импорта:** Встроенный загрузчик моделей форматов Minecraft Bedrock (геометрия + UV) и Java Edition (версии до 1.21.5).
* **Физика:** Интеграция `rapier3d` для базовых коллизий и гравитации.
* **Логирование:** Трёхуровневая система логов (IS_DEBUG, IS_FLOW_DEBUG, IS_EXT_DEBUG) с форматом `[Класс][Метод] — сообщение`.
