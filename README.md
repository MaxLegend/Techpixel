
# Techpixel Project Documentation

**Version:** 0.0.1 (Alpha)

## Overview

Techpixel is a voxel-based 3D game engine inspired by the Minecraft architecture. The project originated from the idea of leveraging Large Language Models (LLMs) to create a functional game engine, effectively democratizing complex development for those without deep coding expertise.

### The "No-Code" Challenge

The engine was developed over a period of 4 months with a strict constraint: **no manual code entry.** The primary goal was to utilize LLMs (Claude, GLM-5/5.1) for all programming tasks while using the Rust programming language as the foundation. Rust was selected for its robust documentation and extensive library ecosystem, which are critical for generating precise, context-aware AI prompts.

## Technical Specifications & Features

### 0. Rendering Pipeline

The engine follows a specific sequence for rendering passes:
`Sky` → `World Geometry` → `Entity Geometry` → `Transparent Blocks` → `Outline` → `UI`.

**Machine State Architecture:**
`push()` → `load()` → `start()` → `[update() / render() / build_ui()]` → `unload()`

### 1. Global Illumination & Lighting

* **Systems:** Voxel Cone Tracing (VCT) for global illumination.
* **Shadows:** DDA voxel raycast for shadow mapping.
* **Advanced Features:**
* Dynamic lighting support (e.g., light sources attached to entities).
* Transparent rendering pass with partial occlusion and colored shadows (e.g., light passing through red glass).
* Volumetric rays and ambient halos around light sources.



### 2. Configuration & World Generation

* **JSON-Driven:** A flexible configuration system allows for real-time adjustment of engine parameters, including chunk sizes.
* **Infinite Verticality:** Supports infinite digging and building. The engine utilizes a layered world generation system.
* **Biome Pipeline:**
* *Geographic Layer:* Continents, islands, mountains.
* *Climatic Layer:* Z-axis climate zones, humidity.
* *Geological Layer:* Magmatic, metamorphic, and sedimentary regions.
* *Stratigraphic Layer:* Preset depth layers (grass, dirt, clay, stone, bedrock).


* **Integrated Visualizer:** An in-game GUI allows for real-time adjustments to world generation parameters, with a "Save to JSON" function to persist settings.

### 3. Block Management

* **Block Editor:** A dedicated in-game interface for creating and modifying blocks.
* **PBR & Texturing:** Supports PBR materials, light emission, and advanced texture splatting (up to 5 sampling layers, allowing for blending textures like grass over dirt).

### 4. Engine Infrastructure & Optimization

* **Optimization:**
* Frustum culling (by chunk and geometry).
* Multi-threaded chunk loading/unloading.
* Directional priority rendering.
* 3-level Level of Detail (LOD) system.


* **Debugging:** Comprehensive runtime monitoring via `F3` (Standard Debug) and `F4` (Profiler).
* **Import Support:** Native loader for Minecraft Bedrock (geometry + UV) and Java Edition (pre-1.21.5) model formats.
* **Physics:** Integrated `rapier3d` for basic collisions and gravity.
* **Logging:** Three-tier logging system (IS_DEBUG, IS_FLOW_DEBUG, IS_EXT_DEBUG) formatted as `[Class][Method] — message`.
